# 01-개선점

프로젝트 분석 기준으로 정리한 개선 제안 목록입니다. 우선순위·난이도는 팀 상황에 맞게 조정하면 됩니다.

---

## 1. 도메인·일관성

| 항목 | 현재 | 제안 |
|------|------|------|
| **Memo.id 타입** | `String` | MongoDB `ObjectId` 사용 시 `ObjectId` 또는 `@Id` + `ObjectId`로 통일하면 `mongoFindOneMemo(ObjectId id)` 등과 일치시킬 수 있음. |
| **Users 모델** | `Long id` 사용, `@Id` 없음 | `@Id` 지정 및 MongoDB에서 자동 생성 여부 정리. 필요 시 `ObjectId` 기반으로 통일 검토. |
| **컬렉션 명칭** | `user`, `user2`, `memo` | 역할이 다르면 `users`(게시글성), `accounts`(회원), `memos` 등으로 구분해 가독성 향상. |

---

## 2. 보안

| 항목 | 제안 |
|------|------|
| **CSRF** | Spring Security 기본 CSRF는 활성화 유지. Thymeleaf 폼에는 `_csrf` 토큰 포함 확인. |
| **비밀번호** | 이미 BCrypt 사용 중. 정책(길이·복잡도) 검증은 회원가입/비밀번호 변경 시 추가 가능. |
| **세션/쿠키** | 운영 시 `server.servlet.session.cookie.http-only`, `secure` 등 설정 검토. |
| **관리자 계정** | 최초 ADMIN 계정 생성용 스크립트 또는 Flyway/초기화 로직 있으면 운영에 유리. |

---

## 3. 코드 구조·유지보수

| 항목 | 현재 | 제안 |
|------|------|------|
| **UserController** | 메모·유저·이미지 업로드 로직이 한 컨트롤러에 혼재 | Memo 전용 `MemoController`, 이미지 업로드·썸네일은 `ImageService` 등으로 분리해 단일 책임 유지. |
| **썸네일/이미지 처리** | 컨트롤러 내부에 BufferedImage·리사이즈 코드 존재 | Thumbnailator 등은 `ImageService`(또는 전용 유틸)로 이동해 재사용·테스트 용이하게. |
| **예외 처리** | ResponseEntity 등 일부만 사용 | `@ControllerAdvice` + 공통 에러 DTO로 4xx/5xx 응답 형식 통일. |
| **주석/미사용 코드** | `// @PostMapping`, `// System.out.println` 등 | 정리하거나 필요한 부분만 주석으로 남기기. |

## 4. 설정·운영

| 항목 | 제안 |
|------|------|
| **application.yml** | `application-sample.yml`만 있는 상태 | 로컬용 `application.yml` 또는 `application-local.yml` 예시 추가. 실제 URI/비밀은 환경변수나 profile로 분리. |
| **로깅** | 기본 로깅 | `logging.level.com.myMongoTest: DEBUG` 등 패키지별 레벨 설정으로 디버깅 편의 확보. |
| **테스트** | 기본 테스트 클래스 존재 | MongoDB Testcontainers 또는 내장 MongoDB로 통합 테스트 추가 시 CI/리팩터 시 안정성 향상. |

---

## 5. API·뷰

| 항목 | 제안 |
|------|------|
| **REST 스타일** | 일부 `@ResponseBody` + ResponseEntity 혼용 | 이미지·메모 등은 REST 리소스 경로(`/api/memos`, `/api/images`)로 구분하면 클라이언트 연동 시 명확. |
| **에러 메시지** | 한글 메시지 직접 노출 | 메시지 키 기반(properties) 사용 시 다국어·수정이 쉬움. |
| **파일 확장자 검증** | 동영상 확장자 하드코딩 (`mp4`, `mov` 등) | 허용 확장자/Content-Type을 상수 또는 설정으로 두고 검증 로직 통일. |
