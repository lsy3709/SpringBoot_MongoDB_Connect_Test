# 03-로그인 리다이렉트 오류 수정 및 기본 관리자 계정

로그인/회원가입 후 리다이렉트 시 발생하던 오류의 원인과 조치, 기본 관리자 계정 추가 내용을 정리한 문서입니다.

---

## 1. 현상

- **회원 가입** 또는 **로그인** 후 목적지로 리다이렉트될 때 오류가 발생하고, **새로고침하면 정상 동작**하는 현상이 있었음.

---

## 2. 원인 분석

### 2.1 로그인 성공 직후 리다이렉트 오류

- 로그인 폼 제출 후 Spring Security가 `defaultSuccessUrl("/admin")` 등으로 **302 리다이렉트**를 보냄.
- 이때 **Set-Cookie(JSESSIONID)** 와 **Location: /admin** 이 같은 응답에 실리는데, 일부 브라우저/환경에서는 리다이렉트 요청(GET /admin)에 **세션 쿠키가 포함되지 않는 경우**가 있음.
- 그 결과 GET /admin 요청이 **비인증**으로 처리되어 401 등 오류가 발생하고, 사용자는 오류 화면을 보게 됨.
- **새로고침** 시에는 이미 쿠키가 저장된 상태라 GET /admin이 인증된 요청으로 가서 정상 동작함.

### 2.2 회원가입 후 처리

- 회원가입 성공 시 `redirect:/` 로 보내고 있었고, **중복 이메일**일 때는 뷰 이름 `"/joinForm"`만 반환해 **리다이렉트가 아닌 포워드**처럼 동작하는 문제가 있음.
- joinForm 템플릿은 `th:object="${user}"` 를 사용하는데, 컨트롤러에서는 `"User2"` 로 모델을 넣어 **바인딩 불일치** 가능성이 있음.

---

## 3. 조치 내용

### 3.1 기본 관리자 계정 (admin / admin1234)

| 항목 | 내용 |
|------|------|
| **목적** | 최초 기동 시 관리자 계정이 없으면 자동 생성 |
| **구현** | `config/DefaultAdminInitializer.java` (ApplicationRunner) |
| **계정** | 이메일 `admin`, 비밀번호 `admin1234`, 역할 `ADMIN` |
| **동작** | 기동 시 `user2` 컬렉션에 `email == "admin"` 이 없을 때만 1회 생성, BCrypt로 비밀번호 저장 |

### 3.2 로그인 성공 후 리다이렉트 오류 해결

| 항목 | 내용 |
|------|------|
| **방식** | 로그인 성공 시 **즉시 /admin으로 보내지 않고**, 중간 경유 URL로 한 번 리다이렉트한 뒤, 해당 페이지에서 다시 목적지로 리다이렉트 |
| **흐름** | `POST /login` → (성공) → `302 /login/redirect?target=/admin` (이 응답에 Set-Cookie) → 브라우저 `GET /login/redirect?target=/admin` (쿠키 포함) → 서버에서 `302 /admin` → 브라우저 `GET /admin` (쿠키 포함) → 정상 렌더링 |
| **구현** | `LoginRedirectAuthenticationSuccessHandler`: 로그인 성공 시 `/login/redirect?target=...` 로 보냄. `SecurityConfig`에서 formLogin에 이 핸들러 적용. `/login/redirect` 는 인증 필요. `UserController#loginRedirect`: `target` 쿼리 파라미터가 있으면 해당 **상대 경로만** 허용해 리다이렉트(오픈 리다이렉트 방지), 없으면 `/admin` |

### 3.3 회원가입 관련 수정

| 항목 | 내용 |
|------|------|
| **모델 키** | joinForm에서 사용하는 `th:object="${user}"` 에 맞춰 컨트롤러에서 `model.addAttribute("user", new User2())` 로 통일 |
| **중복 이메일** | 이미 가입된 이메일이면 `redirect:/joinForm` + flash 메시지 `joinErrorMsg` 로 안내 |
| **비밀번호 공란** | 비밀번호가 비어 있으면 `redirect:/joinForm` + flash 메시지로 안내 |
| **가입 성공** | 가입 성공 시 `redirect:/login` + flash 메시지 `joinSuccessMsg` ("회원가입이 완료되었습니다. 로그인해 주세요.") |
| **뷰** | joinForm.html에 `joinErrorMsg` 표시, loginForm.html에 `joinSuccessMsg` 표시 추가 |

## 4. 변경·추가된 파일 요약

| 구분 | 파일 | 설명 |
|------|------|------|
| **추가** | `config/DefaultAdminInitializer.java` | 기동 시 기본 관리자(admin/admin1234) 생성 |
| **추가** | `config/LoginRedirectAuthenticationSuccessHandler.java` | 로그인 성공 시 `/login/redirect?target=...` 로 보내는 핸들러 |
| **수정** | `config/SecurityConfig.java` | formLogin에 successHandler 적용, `/login/redirect` 인증 필요로 설정 |
| **수정** | `controller/UserController.java` | `GET /login/redirect` 처리, joinForm/joinUser 모델·리다이렉트·flash 메시지 정리 |
| **수정** | `templates/joinForm.html` | `joinErrorMsg` 표시 |
| **수정** | `templates/loginForm.html` | `joinSuccessMsg` 표시 |

---

## 5. 사용 시 참고

- **기본 관리자**: 첫 실행 후 `admin` / `admin1234` 로 로그인 가능. 이미 `admin` 이메일이 있으면 생성하지 않음.
- **로그인 후**: 로그인 성공 시 `/login/redirect` 를 거쳐 `/admin`(또는 저장된 요청 URL)으로 이동하므로, 새로고침 없이 정상적으로 관리자 페이지가 열림.
- **회원가입 후**: 가입 완료 시 로그인 페이지로 이동하며, flash 메시지로 "회원가입이 완료되었습니다. 로그인해 주세요."가 표시됨.
