# AWS Lightsail 메모리 절약 전략 (1GB RAM 환경)

1 GB RAM, 1 vCPU, 40 GB SSD 사양에서 Spring Boot + MongoDB를 같은 머신에 운영할 때 메모리를 아끼는 방법을 정리한 문서입니다.

---

## 1. 현재 상황 가정

| 항목 | 값 |
|------|-----|
| **인스턴스** | 1 GB RAM, 1 vCPU, 40 GB SSD |
| **평상시 사용** | CPU 1% 미만, Mem 약 733M / 970M |
| **구성** | Spring Boot 앱 + Docker MongoDB (동일 머신) |

1GB 중 대부분이 사용 중이므로, 트래픽 증가나 피크 시 OOM(Out of Memory)을 막기 위해 **JVM·MongoDB·OS** 쪽 메모리를 나누어 제한하는 것이 좋습니다.

---

## 2. JVM 힙 제한 (가장 중요)

Spring Boot는 기본 힙을 넉넉히 쓰려 하므로, **1GB 머신에서는 반드시 상한을 지정**해야 합니다.

### 권장 예시 (Spring Boot + MongoDB 둘 다 돌릴 때)

- **Spring Boot JVM**: 힙 **최대 384M ~ 512M**
- **나머지**: OS + MongoDB + 버퍼용

### 적용 방법

**방법 A – systemd 서비스 파일에서**

```ini
[Service]
# 힙 최대 384MB, 최소 256MB (너무 작으면 GC 과다 발생)
Environment="JAVA_OPTS=-Xms256m -Xmx384m -XX:+UseG1GC"
ExecStart=/usr/bin/java $JAVA_OPTS -jar /path/to/your-app.jar
```

**방법 B – JAR 실행 시 직접**

```bash
java -Xms256m -Xmx384m -XX:+UseG1GC -jar build/libs/SpringBoot_MongoDB_Connect_Test-0.0.1-SNAPSHOT.jar
```

### 옵션 설명

| 옵션 | 설명 |
|------|------|
| `-Xms256m` | 최소 힙 256MB (시작 시부터 고정해 단편화·리사이즈 비용 감소) |
| `-Xmx384m` | 최대 힙 384MB (이 값을 넘지 않아 OOM 위험 감소) |
| `-XX:+UseG1GC` | G1 가비지 컬렉터 사용 (소형 힙에서도 상대적으로 예측 가능한 지연) |

- 1GB 머신에서 **Spring Boot만** 512MB까지 쓰고 싶다면 `-Xmx512m`까지 올려도 되지만, MongoDB와 나눠 쓸 경우 **384m 권장**.
- `-Xms`와 `-Xmx`를 같게 두면(예: 384m) 힙이 고정되어 메모리 사용이 더 예측 가능해집니다.

---

## 3. Spring Boot / Tomcat 튜닝

내장 톰캣 스레드·커넥션 수를 줄이면 메모리와 CPU를 아낄 수 있습니다.

### application.yml 예시 (선택)

```yaml
server:
  tomcat:
    threads:
      max: 20          # 기본 200 → 20 (동시 요청 적을 때)
      min-spare: 2     # 유휴 스레드 최소 개수
    max-connections: 100   # 동시 커넥션 상한
    accept-count: 10       # 대기 큐
```

- 트래픽이 매우 적다면 `max: 10` 정도로 더 줄여도 됩니다.
- 동시 접속이 많아지면 이 값을 조금씩 올리면서 메모리 사용량을 확인하는 것이 좋습니다.

---

## 4. MongoDB 메모리 제한 (Docker)

같은 1GB 머신에서 MongoDB를 Docker로 띄운다면 **캐시 크기를 제한**해야 합니다. 제한이 없으면 MongoDB가 RAM을 많이 쓰려 합니다.

### 실행 시 옵션

```bash
docker run -d \
  --name mongodb \
  -p 27017:27017 \
  -v mongodb_data:/data/db \
  --memory=400m \
  mongo:6.0 \
  --wiredTigerCacheSizeGB 0.25
```

| 항목 | 설명 |
|------|------|
| `--memory=400m` | 컨테이너가 사용할 수 있는 메모리 상한 (선택이지만 권장) |
| `--wiredTigerCacheSizeGB 0.25` | WiredTiger 캐시를 **256MB**로 제한 (필수에 가깝게 권장) |

- 1GB 머신에서 Spring Boot에 384MB 줄 경우, MongoDB는 **256MB 캐시 + 프로세스 오버헤드** 정도로 두는 것이 안전합니다.
- `--memory=400m`은 “이 컨테이너는 400MB 넘지 말라”는 의미로, OOM 시 컨테이너만 종료되고 호스트는 보호할 수 있습니다.

---

## 5. OS·스왑

- **스왑**: 1GB만으로는 부족할 수 있으므로, 스왑 512MB~1GB 정도 두면 OOM 직전 완충에 도움이 됩니다. (디스크 I/O가 늘어나 성능은 떨어질 수 있음.)
- **불필요한 서비스**는 중지해 두면 메모리를 조금이라도 아낄 수 있습니다.

```bash
# 스왑 확인
free -h
```

---

## 6. 애플리케이션 레벨에서 아끼기

- **대용량 캐시 금지**: 로컬 캐시(예: caffeine, 맵 캐시)를 쓰지 않거나, 사용 시 최대 개수/용량을 작게 제한합니다.
- **연결 풀**: MongoDB 등 DB 연결 풀 크기는 기본값 또는 작게 유지(예: 10~20). 과도한 풀은 메모리를 먹습니다.
- **대용량 목록 조회**: 한 번에 많이 가져오지 말고, **페이지네이션·커서**로 나눠 조회(이미 적용된 방식 유지).
- **파일 업로드**: `spring.servlet.multipart.max-file-size` 등으로 크기 제한 유지.

## 7. 배포 가이드와 연동 (systemd 예시)

`docs/05-AWS-Lightsail-배포-가이드.md`의 systemd 서비스 예시에 메모리 제한을 넣은 형태입니다.

```ini
[Unit]
Description=Spring Boot MongoDB App
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/SpringBoot_MongoDB_Connect_Test

# 메모리 절약: 힙 상한 384MB, G1GC
Environment="JAVA_OPTS=-Xms256m -Xmx384m -XX:+UseG1GC"
ExecStart=/usr/bin/java $JAVA_OPTS -jar /home/ubuntu/SpringBoot_MongoDB_Connect_Test/build/libs/SpringBoot_MongoDB_Connect_Test-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod

Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

- 경로·유저는 실제 환경에 맞게 수정하면 됩니다.

## 8. 메모리 사용량 확인

```bash
# 전체 메모리
free -h

# 프로세스별 메모리 (RSS 기준)
ps aux --sort=-%mem | head -20

# Java 프로세스 힙 요약 (해당 PID에 맞게 숫자 변경)
jcmd <PID> GC.heap_info
```

- Lightsail 대시보드의 **메트릭(CPU, 메모리)** 도 주기적으로 보면서, 위 설정 변경 후 추이를 확인하는 것을 권장합니다.

---

## 9. 요약 체크리스트

| 순서 | 항목 | 권장 설정 |
|------|------|-----------|
| 1 | JVM 힙 | `-Xms256m -Xmx384m` (또는 `-Xmx512m`까지) |
| 2 | GC | `-XX:+UseG1GC` |
| 3 | Tomcat 스레드 | `server.tomcat.threads.max: 20` (필요 시 조정) |
| 4 | MongoDB Docker | `--wiredTigerCacheSizeGB 0.25`, 필요 시 `--memory=400m` |
| 5 | 스왑 | 512MB~1GB (선택) |
| 6 | 모니터링 | `free -h`, `ps aux`, Lightsail 메트릭 |

이 구성을 적용하면 1GB RAM 환경에서 Spring Boot와 MongoDB가 서로 메모리를 나눠 쓰면서, 평상시 사용량을 줄이고 OOM 위험을 낮출 수 있습니다.
