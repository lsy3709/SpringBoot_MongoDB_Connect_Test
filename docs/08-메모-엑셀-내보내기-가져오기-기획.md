# 메모 데이터 엑셀 내보내기·가져오기 기능 기획

## 1. 개요

- **목적**: 메모 데이터를 외부 파일(엑셀)로 내보내고, 엑셀 파일에서 다시 가져올 수 있게 한다.
- **이미지**: 엑셀 시트에 이미지를 첨부(임베드)하여 내보내기·가져오기 지원.

---

## 2. 현재 메모 구조 (Memo)

| 필드 | 타입 | 비고 |
|------|------|------|
| id | String | MongoDB ObjectId |
| title | String | 제목 |
| message | String | 메모 내용 |
| dateField | String | 등록일 (yyyy-MM-dd HH:mm:ss) |
| imageFileObjectId | String | GridFS ObjectId |
| imageFileName | String | 이미지 파일명 (GridFS) |
| categoryId | String | 카테고리 ID |
| expiryDate | String | 유통기한 (yyyy-MM-dd) |
| tags | List\<String\> | 태그 목록 |

- 이미지는 **GridFS**에 저장되며, `imageFileName`으로 참조.

---

## 3. 엑셀 내보내기 (Export)

### 3.1 포맷

- **파일 형식**: `.xlsx` (Apache POI 사용)
- **시트**: 1개 시트, 1행 = 헤더, 2행~ = 메모 1건씩

### 3.2 컬럼 구성

| 컬럼 | 엑셀 헤더 | 내용 |
|------|-----------|------|
| A | 제목 | title |
| B | 메모 | message |
| C | 등록일 | dateField |
| D | 유통기한 | expiryDate |
| E | 태그 | tags를 쉼표로 연결 (예: 과일,냉동) |
| F | 카테고리ID | categoryId |
| G | 이미지 | **이미지 셀에 그림 삽입** (선택) 또는 이미지 파일명만 |

### 3.3 이미지 처리 (내보내기)

- **방법 A (권장)**: 해당 행의 이미지 셀에 Apache POI `ClientAnchor` + `Drawing`으로 이미지 삽입.
  - GridFS에서 `imageFileName`으로 바이트 조회 → 워크시트에 그림으로 추가.
- **방법 B**: 이미지 컬럼에는 파일명만 기입. 실제 이미지는 **ZIP 패키지**로 함께 내보내기 (예: `메모내보내기_20250310.zip` 안에 `data.xlsx` + `images/` 폴더).

기획 단계에서는 **방법 A(엑셀 셀에 이미지 임베드)** 를 기준으로 하면, 파일 하나로 관리 가능.

### 3.4 API·UI

- **API**: `GET /admin/export/memos.xlsx` (또는 `/api/memos/export`)
  - 인증: ADMIN, ADUSER
  - 응답: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, 파일 다운로드
- **UI**: 인벤토리(admin.html) 또는 대시보드에 "엑셀 내보내기" 버튼 추가.

---

## 4. 엑셀 가져오기 (Import)

### 4.1 입력 포맷

- 내보내기와 **동일한 컬럼 구조**의 xlsx 파일.
- 이미지: 같은 시트의 해당 셀에 넣었던 이미지가 있으면 바이트로 읽어 GridFS에 저장.

### 4.2 처리 흐름

1. 업로드된 xlsx 파싱 (Apache POI).
2. 2행부터 순회:
   - 제목·메모·등록일·유통기한·태그·카테고리ID 매핑.
   - 해당 행의 이미지 셀에서 그림 추출 → 바이트로 저장 → `ImageService.storeFromBytes()` 등으로 GridFS 저장 → `imageFileName` 설정.
3. 메모는 **새로 삽입** (id 생성). 기존 메모 수정이 아닌 **추가**로 처리할지, 아니면 id 매칭으로 업데이트할지는 정책 결정.

### 4.3 API·UI

- **API**: `POST /admin/import/memos` (multipart/form-data, file=xlsx)
  - 인증: ADMIN, ADUSER (또는 ADMIN만)
  - 응답: 성공 시 "N건 가져옴", 실패 행 번호·에러 메시지 등.
- **UI**: 인벤토리 또는 대시보드에 "엑셀 가져오기" 버튼 + 파일 선택 → 업로드.

---

## 5. 기술 요구 사항

### 5.1 의존성

```gradle
// build.gradle
implementation 'org.apache.poi:poi-ooxml:5.2.5'  // xlsx 읽기/쓰기, 이미지 삽입
```

- POI로 시트에 이미지 넣기: `XSSFSheet.createDrawingPatriarch()`, `ClientAnchor`, `addPicture(byte[], format)`.

### 5.2 서비스·컨트롤러

| 구분 | 내용 |
|------|------|
| **MemoExcelService** (신규) | `byte[] exportToExcel(List<Memo>)`, `ImportResult importFromExcel(InputStream)` |
| **ImageService** 확장 | `byte[] getImageBytesByFilename(String filename)` (GridFS → 바이트), `StoredFileInfo storeFromBytes(byte[], String filename)` (가져오기용) |
| **MemoExportController** 또는 **AdminMemberController** 확장 | `GET /admin/export/memos.xlsx`, `POST /admin/import/memos` |

### 5.3 보안

- 내보내기/가져오기 URL은 `hasAnyRole("ADMIN","ADUSER")` 또는 `hasRole("ADMIN")` 적용.
- 가져오기 시 파일 크기 제한 (예: 10MB), 행 수 제한 (예: 1000행)으로 DoS 방지.

---

## 6. 구현 순서 제안

1. **의존성 추가**: `poi-ooxml`
2. **ImageService**: `getImageBytesByFilename()`, (필요 시) `storeFromBytes()` 추가
3. **MemoExcelService**: 엑셀 생성(이미지 셀 포함), 엑셀 파싱(이미지 추출) 구현
4. **컨트롤러**: 내보내기 GET, 가져오기 POST
5. **UI**: 인벤토리/대시보드에 "엑셀 내보내기", "엑셀 가져오기" 버튼 및 가져오기 폼

---

## 7. 정리

- **가능 여부**: 엑셀(xlsx)로 메모 내보내기·가져오기 **가능**하며, **이미지를 셀에 첨부**하는 방식도 Apache POI로 구현 가능.
- **선택 사항**: 이미지를 엑셀 셀에 넣을지, 별도 ZIP으로 이미지 폴더를 둘지만 정하면 됨. 위안에서는 셀 임베드(방법 A)를 기준으로 기획함.
