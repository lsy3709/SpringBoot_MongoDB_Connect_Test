# 작업계획 체크리스트

> Spring Boot 3 + MongoDB + Thymeleaf + Spring Security 스택 기준

---

## 1. 메모리 최적화 (Memory Optimization)

### MongoDB Connection Pool
- [x] `spring.data.mongodb` connection pool 설정 (`maxPoolSize`, `minPoolSize` URI 파라미터)
- [x] 적절한 `maxIdleTimeMS` 설정 (prod: 60000, default: 30000)
- [x] 연결 풀 모니터링 (Actuator health, MongoDB 상태 포함)

### 캐시 설정
- [x] Spring Cache (`@EnableCaching`) 적용 (SpringCacheConfig)
- [x] 자주 조회되는 도메인(카테고리)에 `@Cacheable` 적용
- [x] Caffeine 의존성 추가 및 설정
- [x] 캐시 만료 TTL 및 eviction 정책 정의 (10분)

### 대용량 리스트 처리
- [x] findAll·큰 쿼리 → max limit 적용 (app.memo.findAll-max-size: 500)
- [x] `Pageable` / `Slice` 사용 (MemoRepository, /findAllMemoPageable)
- [ ] 불필요한 `fetch`·`projection` 최소화
- [ ] Stream 기반 처리 시 메모리 사용량 검토

### JVM 힙 튜닝
- [x] `-Xmx`, `-XX:MetaspaceSize` 프로덕션용 설정 (build.gradle `bootRun -Pprod`)
- [x] 1GB 메모리 환경: `-Xmx512m` ~ `-Xmx768m` 수준 적용
- [x] GC 로그 설정 (`-Xlog:gc*:file=./build/gc-%t.log`, bootRun -Pprod)

### Tomcat·기타
- [x] `server.tomcat.threads.max` 적정값 확인 (default/prod 50)
- [x] `spring.servlet.multipart.max-file-size` 10MB 유지 (default/prod)

## 4. 보안 관련 검사 및 작업

### OWASP Top 10 대응
- [x] A01 Broken Access Control – 권한·역할 체크 (SecurityAccessControlTest, /admin hasRole ADMIN)
- [ ] A02 Cryptographic Failures – 민감 데이터 암호화
- [x] A03 Injection – NoSQL/ReDoS 방지 (RegexEscape로 사용자 입력 이스케이프)
- [ ] A04 Insecure Design – 설계 단계 보안 고려
- [x] A05 Security Misconfiguration – 프로덕션 설정 점검 (prod: secure cookie, INFO 로그, threads 50)
- [ ] A06 Vulnerable Components – 의존성 취약점 스캔
- [x] A07 Auth Failures – 인증·비밀번호 정책 (PasswordPolicyValidator, BCrypt strength 10)
- [ ] A08 Software Integrity – 빌드·배포 파이프라인 신뢰
- [x] A09 Logging/Monitoring – 보안 이벤트 로깅 (미인증 접근, 로그인 실패)
- [ ] A10 SSRF – 외부 URL 요청 검증

### 이미 처리된 항목 (확인·유지)
- [x] CSRF (Spring Security 기본, form 로그인·POST에 토큰 적용)
- [x] XSS (Thymeleaf 기본 HTML 이스케이프)
- [x] SQLi (MongoDB 사용, SQL 미사용. NoSQL Injection: RegexEscape, 바인딩 쿼리)

### 비밀번호 정책
- [x] 최소 길이·복잡도 정책 적용 (PasswordPolicyValidator)
- [x] BCrypt 스트렝스 확인 (strength 10, PasswordEncoderConfigTest)
- [ ] 비밀번호 변경·만료 정책 (필요 시)

### 세션
- [x] `httpOnly`, `secure`, `same-site` 쿠키 설정 유지
- [x] 세션 타임아웃 30분 설정 (default, prod)

### HTTPS
- [ ] 프로덕션 HTTPS 적용
- [ ] HSTS 설정 검토

### 의존성 취약점 스캔

- [ ] NVD API 키 설정 후 실행 (403/404 해결: https://nvd.nist.gov/developers/request-an-api-key)
- [ ] Dependabot·Renovate 등 자동 업데이트 검토

### CORS
- [x] CorsConfigurationSource 빈 설정, SecurityConfig에 `.cors(withDefaults())` 적용
- [x] `app.cors.allowed-origins` 프로퍼티로 허용 origin 제어 (기본 *)

## 5. 점진적 진행 (Phased Approach)

### 우선순위 및 의존성

| 우선순위 | 작업 | 의존성 | 예상 난이도 |
|---------|------|--------|------------|
| P0 | 메모리 최적화 (연결 풀, 힙, 페이지네이션) | 없음 | 중 |
| P0 | 보안 (의존성 스캔, CORS, Rate Limiting) | 없음 | 중 |
| P1 | 메모리 캐시 도입 | MongoDB 안정화 후 | 중 |
| P1 | 비밀번호·세션 정책 강화 | Spring Security 설정 | 하 |
| P2 | @Async·비동기 엔드포인트 | P0/P1 완료 후 | 중 |
| P2 | CompletableFuture·배치 처리 | 비동기 설정 완료 후 | 중 |
| P3 | 메시지 큐 도입 | 필요성 검토 후 | 상 |
| P3 | Reactive MongoDB (선택) | 아키텍처 결정 후 | 상 |

### Phase 1 – 기반 안정화 (1~2주)
1. MongoDB connection pool 설정
2. JVM 힙·GC 튜닝
3. 대용량 리스트 페이지네이션 적용
4. OWASP Dependency-Check 실행 및 의존성 업데이트
5. CORS·Rate Limiting 기본 적용

### Phase 2 – 성능·보안 강화 (2~3주)
1. Spring Cache 도입 및 주요 조회 캐시 적용
2. 비밀번호 정책·세션 강화
3. OWASP Top 10 체크리스트 점검
4. `@Async` 도입 및 용도 정의

### Phase 3 – 고급 최적화 (선택)
1. CompletableFuture·배치 처리 적용
2. 메시지 큐 도입 검토
3. Non-blocking I/O·Reactive 전환 검토
